# Product

## Register

brand

## Users

Fernwaerts serves people who want a useful location history without handing
their movement data to a third-party platform. The public homepage and docs
need to speak to privacy-conscious app users, self-hosters, and open-source
contributors who are evaluating whether the project is trustworthy enough to
run and practical enough to set up.

## Product Purpose

Fernwaerts is a privacy-focused, self-hosted location history product. It helps
people keep, inspect, and understand their own location timeline through a
Flutter app backed by local-first storage and a self-hosted Supabase stack.
Success means the public site makes the app feel approachable and transparent,
the docs make setup feel achievable, and both surfaces reinforce that the user
keeps control of their data.

## Brand Personality

Open, fun, private. The brand should feel candid about how the system works,
light enough to make personal history feel pleasant, and careful enough to
earn trust around sensitive location data. The homepage should borrow loosely
from the app's visual language rather than feeling like a separate marketing
template.

## Anti-references

Fernwaerts should not feel like a crypto or web3 app, and it should not look
like a generic SaaS landing page. Avoid visual patterns that imply surveillance,
growth analytics, enterprise monitoring, speculative finance, or a polished
but interchangeable startup template.

## Design Principles

- Lead with user ownership: every major surface should make it clear that the
  user's journey and data remain theirs.
- Let the app set the tone: homepage visuals should echo the Flutter app's map,
  calendar, translucent surfaces, soft green primary color, and personal
  timeline feel.
- Make privacy understandable: explain sensitive setup and data flow in plain
  language, with direct docs paths instead of vague reassurance.
- Keep it playful without losing trust: use motion, imagery, and warm copy to
  make location history feel personal, while keeping authentication, hosting,
  and sync copy precise.
- Prefer concrete proof over claims: show app screenshots, setup steps, open
  source context, and real product states instead of abstract feature cards.

## Accessibility & Inclusion

Target accessible, responsive public pages and documentation with readable
contrast, keyboard-reachable controls, descriptive alt text for meaningful
images, and reduced-motion alternatives for decorative or atmospheric motion.
Because location history is sensitive, avoid exposing personal location data,
device identity, tokens, or server details in public examples and logs.
